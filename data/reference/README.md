# Club reference data (constructed stand-ins)

These two files stand in for documents Leyton Orient would normally provide. We do not
have the club's real wage framework or player identity documents, so these are built and
clearly labelled, designed to be swapped for the club's real files when available.

Generated by `python -m lofc.store.reference_data`. Loaded by `python -m lofc.store.load`.

## wage_framework.csv
The most LOFC would pay per week, by position group and age band. Drives affordability.

Provenance:
- **Fact:** the EFL rule that League One wages are capped at 50% of turnover, and LOFC's
  ~£7.7m turnover (2023/24 published accounts). Together this gives a ~£3.5-4m wage pool,
  which the whole grid sits under.
- **Estimate:** the top figure (~£6,500/week for a prime attacker) matches LOFC top-earner
  estimates from Capology and SalarySport (2024/25-2025/26). These sites model wages; they
  are not official, so treat exact numbers as approximate.
- **Assumption:** the position shape (attackers cost most, goalkeepers/full-backs least) and
  the age curve (peak 25-29, lower for under-21s and over-32s). No public EFL positional
  wage data exists, so this is reasoned, not sourced.
- Year note: the budget is present-day (2024-26). The demo player data is 2015/16, so we do
  not make literal affordability claims; on real current data the years align.

## identity_profiles.csv
For each position: which metrics matter (weight) and the minimum standard to be "on profile"
(min_percentile). Weight drives the Phase 4 score; min_percentile drives the Phase 7 filter.

Provenance:
- **Assumption (entirely):** a football-reasoning construction, not a club document. The chosen
  identity is a hard-working, progressive, press-resistant team. Weights sum to 1.0 per position.
- Swap this file for the club's real recruitment profile to change what "good" means.
